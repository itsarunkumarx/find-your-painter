## 1. PWA Setup

> ✅ **Already configured** — the following files handle everything automatically.

### manifest.json (auto-generated by vite-plugin-pwa)

The manifest is generated from `vite.config.js` during build. Output example:

```json
{
  "name": "Find Your Painter",
  "short_name": "PainterApp",
  "description": "Find professional painters for your home",
  "theme_color": "#ffffff",
  "background_color": "#ffffff",
  "display": "standalone",
  "start_url": "/",
  "id": "/",
  "icons": [
    { "src": "icons/icon-192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "icons/icon-512.png", "sizes": "512x512", "type": "image/png" },
    { "src": "icons/icon-512.png", "sizes": "512x512", "type": "image/png", "purpose": "any maskable" }
  ]
}
```

### Service Worker (`src/sw.js`)

Uses Workbox with these caching strategies:
- **Precache** — all Vite build assets (`self.__WB_MANIFEST`)
- **StaleWhileRevalidate** — Google Fonts, scripts, styles
- **CacheFirst** — images (30-day expiry, max 60 entries)
- **Push Notifications** — handles incoming call notifications

### Key Files

| File | Purpose |
|------|---------|
| `vite.config.js` | PWA plugin config with `injectManifest` strategy |
| `src/sw.js` | Service worker with Workbox caching |
| `src/main.jsx` | Registers SW + captures `beforeinstallprompt` |
| `index.html` | PWA meta tags, apple-touch-icon |
| `public/icons/` | 192×192 and 512×512 app icons |

## 2. Install Prompt Flow

```
Browser loads site
  → Service Worker registers
  → manifest.json detected
  → Chrome fires `beforeinstallprompt` event
  → main.jsx captures it → stores in window.deferredPrompt
  → User clicks "Install App" button
  → deferredPrompt.prompt() shows native install dialog
  → User accepts → app installs to home screen
```

The install button logic is in:
- `src/pages/EntryPage.jsx` — hero section buttons
- `src/components/InstallApp.jsx` — floating bottom prompt

## 3. Device Detection

The `src/hooks/useDeviceDetect.js` hook provides:

| Property | Description |
|----------|-------------|
| `isAndroid` | Android user agent detected |
| `isIOS` | iPhone / iPad / iPod detected |
| `isDesktop` | Not a mobile device |
| `isMobile` | Any mobile device |
| `isStandalone` | Already running as installed PWA |

### Button Visibility Rules

| Device | Install App (PWA) | Download APK |
|--------|:-:|:-:|
| Android Mobile | ✅ | ✅ |
| Desktop (Chrome/Edge) | ✅ (when prompt fires) | ❌ |
| iOS (Safari) | ✅ (with instructions) | ❌ |
| Already installed | ❌ | ❌ |

## 4. Capacitor Android APK

### Prerequisites (Manual Steps)

1. **Install Android Studio**: https://developer.android.com/studio
2. **Install Java JDK 17+**: Android Studio usually bundled
3. **Set up Android SDK**: Android Studio → SDK Manager → install API 33+

### Setup Commands

The Capacitor packages are already installed. If starting fresh:

```bash
# Navigate to frontend
cd frontend

# Install Capacitor (already done)
npm install @capacitor/core @capacitor/cli @capacitor/android

# Initialize (already done — capacitor.config.json exists)
npx cap init "Find Your Painter" "com.findyourpainter.app" --web-dir=dist

# Build the web app first
npm run build

# Add Android platform
npx cap add android

# Sync web assets to Android
npx cap sync android
```

### capacitor.config.json

```json
{
  "appId": "com.findyourpainter.app",
  "appName": "Find Your Painter",
  "webDir": "dist",
  "server": {
    "url": "https://find-your-painter.vercel.app",
    "cleartext": true
  },
  "plugins": {
    "SplashScreen": {
      "launchShowDuration": 2000,
      "backgroundColor": "#1a1a2e"
    }
  }
}
```

> **Note:** The `server.url` makes the app load your deployed website. Remove it to bundle assets locally instead.

## 5. Building the APK

### Debug APK (for testing)

```bash
# 1. Build web app
npm run build

# 2. Sync to Android
npx cap sync android

# 3. Open in Android Studio
npx cap open android
```

**In Android Studio:**
1. Wait for Gradle sync to complete
2. Go to **Build → Build Bundle(s) / APK(s) → Build APK(s)**
3. The debug APK will be at:
   ```
   android/app/build/outputs/apk/debug/app-debug.apk
   ```

### Command-line build (without Android Studio UI)

```bash
cd android
./gradlew assembleDebug
```

The APK is at `android/app/build/outputs/apk/debug/app-debug.apk`.

## 6. Signed Release Build

### Step 1: Generate a Keystore

```bash
keytool -genkey -v -keystore findyourpainter-release.keystore -alias findyourpainter -keyalg RSA -keysize 2048 -validity 10000
```

You'll be prompted for a password and your details. **Save the keystore file and password securely!**

### Step 2: Configure signing in Android Studio

1. Open Android Studio → **Build → Generate Signed Bundle/APK**
2. Select **APK**
3. Choose your keystore file, enter alias and passwords
4. Select **release** build variant
5. Click **Finish**

### Or configure via `android/app/build.gradle`:

```groovy
android {
    signingConfigs {
        release {
            storeFile file('findyourpainter-release.keystore')
            storePassword 'YOUR_STORE_PASSWORD'
            keyAlias 'findyourpainter'
            keyPassword 'YOUR_KEY_PASSWORD'
        }
    }
    buildTypes {
        release {
            signingConfig signingConfigs.release
            minifyEnabled true
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        }
    }
}
```

Then build:
```bash
cd android
./gradlew assembleRelease
```

Output: `android/app/build/outputs/apk/release/app-release.apk`

## 7. APK Distribution

### Hosting the APK

1. Rename the built APK to `FindYourPainter.apk`
2. Place it in `frontend/public/FindYourPainter.apk`
3. Deploy your frontend — the APK will be accessible at `https://yourdomain.com/FindYourPainter.apk`

### Download Button (Already Implemented)

The download button in `EntryPage.jsx` and `InstallApp.jsx` points to:
```html
<a href="/FindYourPainter.apk" download="FindYourPainter.apk">
    Download APK
</a>
```

### Android Security Settings

Users will need to:
1. Download the APK
2. Android will warn about "unknown sources"
3. Tap **Settings → Allow from this source**
4. Install the APK

## 8. Testing Guide

### PWA Testing (Chrome DevTools)

1. Open your site in Chrome
2. Press **F12** → **Application** tab
3. Check the following sections:

| Section | What to Verify |
|---------|---------------|
| **Manifest** | All fields loaded, icons visible |
| **Service Workers** | SW registered, status "activated" |
| **Cache Storage** | Precached assets visible |
| **Storage** | Check total usage |

4. Run **Lighthouse** audit → check PWA section for installability

### Service Worker Testing

1. DevTools → **Application → Service Workers**
2. Check "Offline" checkbox
3. Reload the page — the app should still load from cache
4. Uncheck "Offline" to restore

### Install Prompt Testing

1. Open in Chrome (desktop or Android)
2. The install prompt banner should appear (if criteria met)
3. Click "Install App" → native Chrome install dialog appears
4. After installing, the app opens in standalone mode (no browser UI)

**PWA Install Criteria (Chrome):**
- Served over HTTPS
- Has a valid `manifest.json` with `name`, `icons`, `start_url`, `display`
- Has a registered service worker with a `fetch` event handler

### APK Testing on Android

1. Transfer `app-debug.apk` to your Android phone
2. Open the file → allow installation from unknown sources
3. Install and launch
4. Verify the app loads the website correctly
5. Test push notifications, offline behavior

### Mobile Device Testing (without real device)

- Use Chrome DevTools → **Toggle Device Toolbar** (Ctrl+Shift+M)
- Select various devices (Pixel, iPhone, iPad)
- Verify button visibility changes correctly
